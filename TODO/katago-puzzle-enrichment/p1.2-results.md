## Critical Bugs Found & Fixed

### Bug 1: allowMoves JSON format (P0)
- **Symptom:** KataGo analyzing from opponent's perspective
- **Root cause:** Format was `[["B","D4"]]` but KataGo requires `[{"player":"B","moves":["D4"],"untilDepth":1}]`
- **Impact:** All puzzles affected — completely wrong analysis

### Bug 2: Puzzle region from FRAMED position (P0)
- **Symptom:** Puzzle region included frame padding stones
- **Root cause:** Region computed after adding frame stones instead of from original position
- **Impact:** Incorrect board region sent to KataGo

### Bug 3: Broken test fixtures (P0)
- **Symptom:** 6 fixtures had invalid/impossible positions
- **Root cause:** Hastily constructed synthetic SGFs
- **Fix:** Rewrote fixtures: simple_life_death, nakade, double_atari, throw_in, ko_direct, broken_puzzle

### Bug 4: Auto-rejecting high-winrate moves not in top-N (P1)
- **Symptom:** Correct move has winrate 0.92 but gets REJECTED because not in top-5 by visits
- **Root cause:** Validation only checked visit ranking, not winrate
- **Fix:** "Winrate rescue clause" — moves with winrate ≥ 0.7 not in top-N → FLAGGED instead of REJECTED
- **Diagnostic:** Returns `("winrate_rescue"` / `"seki_winrate_rescue")` flag for observability

### Bug 5: extract_correct_first_move ignoring RIGHT markers (P0)
- **Symptom:** Pipeline extracts wrong "correct" move from puzzles where correct variation isn't the first child
- **Root cause:** `extract_correct_first_move()` always picked `root.children[0]`, ignoring SGF RIGHT/CHOICERIGHT comment markers
- **Impact:** 3 puzzles (#19, #27, #30) had correct move in non-first variation → false rejection
- **Fix:** Added `_subtree_has_right_marker()` and `_pick_correct_child()` helpers; all extraction functions now prefer RIGHT-marked variations
- **Verified:** #19 REJECTED→ACCEPTED, #27 FLAGGED→ACCEPTED, #30 REJECTED→FLAGGED

### Bug 6: KataGo config path not passed through DualEngineManager (P1)
- **Symptom:** Custom tsumego config not used
- **Fix:** Added `katago_config_path` parameter through DualEngineManager, CLI functions, CLI parser
