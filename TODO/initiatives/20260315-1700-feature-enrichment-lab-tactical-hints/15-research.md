## 2. Enrichment Lab Deep Audit — Key Findings

### 2A. Detection System (28 detectors)

| Finding | Detail |
|---------|--------|
| **17 of 28 detectors already use board simulation** | Liberty counting, BFS group analysis, pattern matching. Not purely KataGo-signal-based as initially assumed. |
| **DetectionResult.evidence is DISCARDED** | Rich evidence strings (e.g., "Board-state ladder chase: 12 steps, PV-confirmed") are generated by detectors but dropped at TechniqueStage boundary. Only `tag_slug` strings survive to hints/teaching. |
| **Unused KataGo data** | `visit_count` (0/28 detectors use), `root_score` (0/28), `root_winrate` (0/28), `ownership` (4/28), `policy` (6/28), `score_lead` (2/28). Major untapped signal reservoir. |
| **Multi-orientation testing: 0 of 28 detectors** | No rotation/reflection tests exist in any detector test file. Direction-dependent detectors (ladder, net, throw-in) may have orientation bugs. |

### 2B. Hint Generation (3-tier)

| Gap ID | Gap | Current State | Impact |
|--------|-----|---------------|--------|
| HG-1 | **Tier 1 is entirely static** | Just echoes technique name from config. No adaptation to position. | "Snapback (uttegaeshi)" whether 2-stone or 15-stone |
| HG-2 | **Tier 2 is generic** | Uses only `solution_depth` and `refutation_count`. Ignores ownership, policy distribution, PV shape, refutation details. | Generic reasoning: "The solution requires 3 moves of reading" |
| HG-3 | **Tier 3 is formulaic** | Template only varies by technique tag. No contextual variation. | Same template for corner and center |
| HG-4 | **No access to DetectionResult** | `generate_hints()` receives `analysis: dict` + `technique_tags: list[str]`. Evidence strings never reach hints. | Detector's "Move D4 connects 3 groups" → lost |
| HG-5 | **No level-adaptive content** | All levels get identical hint text | Novice and dan player see same reasoning |

### 2C. Teaching Comments

| Gap ID | Gap | Current State |
|--------|-----|---------------|
| TC-1 | **No move intent/instinct** | Layer 1 = technique phrase, Layer 2 = signal phrase. Neither describes WHY the move is played. |
| TC-2 | **Wrong-move comments lack specificity** | 11 condition types exist, but most produce generic text ("Wrong — opponent lives") rather than specific consequences. |
| TC-3 | **PV data available but barely used** | `refutation_classifier.py` receives PV data but classifies into condition strings, losing detail. |

### 2D. Difficulty Estimation

| Gap ID | Gap | Current State |
|--------|-----|---------------|
| DE-1 | **No policy entropy** | Difficulty uses policy prior (correct move strength), structural features, trap density. Not the SPREAD of policy across candidates. |
| DE-2 | **entropy_roi.py exists but for ROI, not difficulty** | Computes ownership entropy for region-of-interest detection. Not wired to difficulty. |

## 4. Prioritized Recommendations

### Tier 1 — Highest Impact, Governance-Endorsed

| F-ID | Feature | Source | Governance Support | Rationale |
|------|---------|--------|-------------------|-----------|
| F-1 | **Policy entropy as difficulty feature** | XR-12, DE-1 | 7/7 unanimous | "Strongest signal in this consultation" — Cho Chikun. ~5 lines of code. Uses existing KataGo data. |
| F-2 | **Pass DetectionResult evidence to hint generator** | XR-14, HG-4 | Internal finding, engineering-endorsed | Currently discarded. Rich evidence like "12-step ladder chase" should feed Tier 2 hints. Zero new detection work. |
| F-3 | **Instinct classification for hints** | XR-1, XR-15 | 7/7 unanimous | Filtered to tsumego-relevant: push, hane, cut, descent, extend. Classify from KataGo policy direction vectors relative to groups. |
| F-4 | **Multi-orientation test infrastructure** | XR-2 | 6/7 approve | `Position.rotate()` / `reflect()` + pytest parametrize. Critical correctness gate for direction-dependent detectors. |

### Tier 2 — Medium Impact, Panel-Endorsed

| F-ID | Feature | Source | Governance Support | Rationale |
|------|---------|--------|-------------------|-----------|
| F-5 | **Level-adaptive hint content** | XR-13, HG-5 | 7/7 unanimous | Gate hint detail by `get_level_category()`. Beginner→consequence, Intermediate→intent, Dan→reading guidance. |
| F-6 | **Top-K rank as observability metric** | XR-6 | 6/7 supplementary | Store `correct_move_rank` in batch metrics. Measure correlation with difficulty before weighting. |
| F-7 | **Expand dead shape catalog** | XR-11 | Not explicitly reviewed | 6→30+ patterns. Public Go knowledge. Low-risk addition. |

### Tier 3 — Low Impact / Deferred
