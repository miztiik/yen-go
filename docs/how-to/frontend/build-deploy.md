## Production Build

### Build Command

```bash
cd frontend
npm run build
```

Output goes to `frontend/dist/`:

```
dist/
├── index.html
├── assets/
│   ├── index-[hash].js    # Main bundle
│   ├── index-[hash].css   # Styles
│   └── ...
└── favicon.ico
```

### Build Configuration

Vite config in `vite.config.ts`:

```typescript
export default defineConfig({
  base: "/yen-go/", // GitHub Pages base path
  build: {
    outDir: "dist",
    sourcemap: true, // Source maps for debugging
    minify: "esbuild", // Fast minification
    rollupOptions: {
      output: {
        manualChunks: {
          // Split vendor chunk for caching
          vendor: ["preact"],
        },
      },
    },
  },
});
```

## Deployment

### GitHub Pages (Automated)

Deployment happens automatically via GitHub Actions on push to `main`:

```yaml
# .github/workflows/deploy.yml
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: 18

      - name: Install and Build
        run: |
          cd frontend
          npm ci
          npm run build

      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./frontend/dist
```

### Manual Deployment

If needed, deploy manually:

```bash
# Build
cd frontend
npm run build

# Deploy (requires gh-pages package)
npx gh-pages -d dist
```

## Build Optimization

### Bundle Analysis

Analyze bundle size:

```bash
# Install analyzer
npm install --save-dev rollup-plugin-visualizer

# Add to vite.config.ts
import { visualizer } from 'rollup-plugin-visualizer';

export default defineConfig({
  plugins: [
    visualizer({
      filename: 'dist/stats.html',
      open: true
    })
  ]
});

# Build and view stats
npm run build
```

### Performance Targets

| Metric         | Target         | How to Check              |
| -------------- | -------------- | ------------------------- |
| Initial bundle | <100KB gzipped | `ls -la dist/assets/*.js` |
| LCP            | <2.5s          | Lighthouse                |
| FCP            | <1.8s          | Lighthouse                |
| TTI            | <3.5s          | Lighthouse                |

### Reducing Bundle Size

```typescript
// ✅ Dynamic imports for code splitting
const PuzzleRush = lazy(() => import("./pages/PuzzleRush"));

// ✅ Tree-shaking friendly imports
import { signal } from "@preact/signals"; // Not: import * as signals

// ❌ Avoid large dependencies
// Don't: import moment from 'moment';
// Do: use native Date or tiny alternative
```

## Caching Strategy

### Static Assets

Vite adds content hashes to filenames for cache busting:

```
# Long-term caching safe
assets/index-a1b2c3d4.js
assets/index-e5f6g7h8.css
```

Configure CDN/hosting for:

- `index.html` → no-cache or short TTL
- `assets/*` → long TTL (1 year)

### Puzzle Data

Puzzles in `yengo-puzzle-collections/` use versioned paths:

```
/sgf/intermediate/2026/01/batch-001/puzzle.sgf
```

## Troubleshooting

### Build Fails

```bash
# Clear caches
rm -rf node_modules/.vite
rm -rf dist

# Reinstall and rebuild
npm ci
npm run build
```

### 404 on GitHub Pages

Ensure `base` is set correctly:

```typescript
// vite.config.ts
export default defineConfig({
  base: "/yen-go/", // Must match repository name
});
```

### Service Worker Issues

```bash
# Clear service worker in browser
# DevTools → Application → Service Workers → Unregister

# Force update
navigator.serviceWorker.getRegistrations().then(regs => {
  regs.forEach(reg => reg.unregister());
});
```

### Large Bundle Size

1. Run bundle analyzer (see above)
2. Check for duplicate dependencies: `npm ls`
3. Use dynamic imports for routes
4. Remove unused dependencies
